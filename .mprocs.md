# mprocs Quick Reference

> Full guide in README.md Development section

## Quick Start

```bash
make install        # First time only
make dev            # Local development
make dev-remote     # Remote backend testing (Docker)
```

## Keyboard Shortcuts

| Key | Action |
|-----|--------|
| `Tab` / `Shift+Tab` | Cycle through processes |
| `Space` | Start/stop process |
| `r` | Restart process |
| `k` | Kill process |
| `f` | Focus process (full screen) |
| `Esc` | Exit focus mode |
| `/` | Search logs |
| `q` | Quit all |

## Common Tasks

### Restart TypeScript Build
1. Press `Tab` until `typescript-watch` is selected
2. Press `r` to restart

### View NextJS Logs Only
1. Press `Tab` until `nextjs-dev` is selected
2. Press `f` for full-screen logs
3. Press `Esc` to exit

### Search Logs
1. Press `/`
2. Type search term
3. Press `Enter`

## Process Descriptions

### Local Mode (`mprocs.yaml`)

- **typescript-watch**: Rebuilds agent-backend on file changes
- **nextjs-dev**: NextJS dev server with hot reload

### Remote Mode (`mprocs.remote.yaml`)

- **typescript-watch**: Same as local
- **remote-mcp-server**: Docker container running MCP server
- **nextjs-remote-mode**: NextJS configured for remote backend

## Troubleshooting

### Process won't start
- Check logs for errors
- Try killing and restarting: `k` then `Space`

### TypeScript changes not appearing
- Restart typescript-watch: Select it, press `r`
- Restart nextjs-dev: Select it, press `r`

### Port conflicts
```bash
# Kill processes using ports
lsof -ti:3000 | xargs kill -9  # NextJS
lsof -ti:3001 | xargs kill -9  # MCP server
```

### Docker issues (remote mode)
```bash
# Rebuild Docker image
make docker-build

# Clean Docker resources
make docker-clean
```

## Configuration Files

- `mprocs.yaml` - Local development config
- `mprocs.remote.yaml` - Remote testing config
- Edit these to add new processes or change settings

## Exit Cleanly

Press `q` to quit - this stops all processes gracefully.

If processes hang, use `Ctrl+C` (may leave orphans).
